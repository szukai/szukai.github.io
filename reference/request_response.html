# QueryResponse

# Feature Properties

If you register a connector method with the feature properties which match following rules, then you need to return a response which is a dict with data mentioned in Attribute section.

Matching wildcard patterns:

  • activities.events.query

# Attributes

Name Type Description Note
response_meta dict Meta data about the response. Must include a 'success' boolean value that indicates whether the request is successful. Typically, if the service only has one activity source, the connector developer can just override activities.events.query to let it raise error when the response is not successful. The default implementation assumes there may be multiple sources so will only return response_meta objects with success set to False. If 'success' is False, additional 'message' and 'error_code' values can be added to specify the error message and status code. Optional attribute ''exhausted': True' may be set to indicate that an activity source doesn't have any more pages.
response object (Optional) The raw requests.Response object. Could be None ifresponse_meta['success'] is False or response_meta['exhausted'] is set and is True.
