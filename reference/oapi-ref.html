# OpenAPI Specification

# Overview

The OpenAPI Specification (formerly Swagger Specification) is an API description format for REST APIs.

In conjunction with the Kloudless specification file, the Kloudless Custom Connector uses an OpenAPI document (yaml) file to describe the connected service's behavior and description.

For examples on how to use the OpenAPI specification with the Kloudless Custom Connector please refer to our Quickstart Guide.

For more information on the OpenAPI specification, check out the OpenAPI website. It can also be found here.

# Things to note

The Kloudless Custom Connector only utilizes certain parts of the OpenAPI specification, a complete specification does not need to be defined or followed.

Under paths, the responses mapping only needs "200" and "description" for parsing:

responses:
    "200":
        description: ok

3

Only the first responses in securityDefinitions will be used. Defining more will be redundant.
