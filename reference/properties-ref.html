# Properties

Each service connected to Kloudless has properties to describe what capabilities and requirements it has. The properties defined below are also used to register the script's connector functions to the appropriate endpoint to facilitate the service's behavior. These endpoints can be found in the Kloudless Developer Documentation

Currently the Custom Connector only supports certain APIs by category:

  • Storage
  • CRM

Properties can be in one of three categories: capabilities, requirements or configuration.

# Authentication

# Capabilities

# auth.account.read

  • Description: Whether the service supports reading account information.

  • Function: The registered function will be called after authentication. In OAuth 2.0, that would be right after the access token is retrieved. In basic auth, it would be after the username and password are received from the pre_auth form.

    For example, in OAuth 2.0, the request_data would always be empty {} here; in basic auth, request_data would be the pre_auth form values like {'username': 'example@example.com', 'password': 'dfk9D{jkl4'}. These values can be validated against account information or used for post-authentication processing.

# Storage

# Capabilities

# storage.files.copy

  • Description: Whether the service supports copying files.
  • Function: Called when the Copy a File endpoint is called to create a new file in the Storage API.

# storage.files.content.update

  • Description: Whether the service supports updating file contents.
  • Function: Called when the Upload a File endpoint is called to an existing file in the Storage API.

# storage.files.create

  • Description: Whether the service supports creating files
  • Function: Called when the Upload a File endpoint is called to create a new file in the Storage API.

# storage.files.create.metadata_response

  • Description: Whether the service supports returning the upload response metadata.
  • Function: Called after the Upload a File endpoint is called to get the new/uploaded file's metadata.

# storage.files.delete

  • Description: Whether the service supports deleting files.
  • Function: Called when the Delete a File endpoint is called to delete a file in the Storage API.

# storage.files.download

  • Description: Whether the service supports downloading file contents.
  • Function: Called when the Download a File endpoint is called to download a file in the Storage API.

# storage.files.thumbnail

  • Description: Whether the service supports reading file thumbnail data.
  • Function: Called when the Download a File's Thumbnail endpoint is called to download a file's thumbnail data in the Storage API.

# storage.files.read

  • Description: Whether the service supports reading file metadata.
  • Function: Called when the Retrieve a File's Metadata endpoint is called to download a file's metadata in the Storage API.

# storage.files.update

  • Description: Whether the service supports updating file metadata.
  • Function: Called when the Update a File's Metadata endpoint is called to update a file's metadata in the Storage API.

# storage.folders.copy

  • Description: Whether the service supports copying folders.
  • Function: Called when the Create a Folder endpoint is called to create a new folder in the Storage API.

# storage.folders.create

  • Description: Whether the service supports creating new folders.
  • Function: Called when the Create a Folder endpoint is called to create a new folder in the Storage API.

# storage.folders.delete

  • Description: Whether the service supports deleting folders.
  • Function: Called when the Delete a Folder endpoint is called to delete a folder in the Storage API.

# storage.folders.list

  • Description: Whether the service supports listing folders and folders contents.
  • Function: Called when the Retrieve Folder Contents endpoint is called to list a folder's contents in the Storage API.

# storage.folders.read

  • Description: Whether the service supports reading folder metadata.
  • Function: Called when the Retrieve Folder Metadata endpoint is called to download a folder's metadata in the Storage API.

# storage.folders.update

  • Description: Whether the service supports updating folder metadata.
  • Function: Called when the Update Folder Metadata endpoint is called to update a folder's metadata in the Storage API.

# storage.links.create.native

  • Description: Whether the service supports creating native links to files or folders.
  • Function: Called when the Create a Link endpoint is called to create a new link in the Storage API.

# storage.files.search

  • Description: Whether the service supports searching in its storage API.
  • Function: Called when the Search endpoint is called to create a new link in the Storage API.
